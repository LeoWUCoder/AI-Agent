## 数组查找功能
```C
#include <stdio.h>
// 实现数组查询功能
int main() {
    int arr[5] = {1, 2, 3, 4, 5};
    int query;
    printf("请输入要查询的数字：");
    scanf("%d", &query);
    
    int found = 0; // 标志变量，表示是否找到
    for (int i = 0; i < 5; i++) {
        if (arr[i] == query) {
            printf("数字 %d 在数组中，索引为 %d\n", query, i);
            found = 1; // 找到后设置标志变量
            break; // 退出循环
        }
    }
    
    if (!found) {
        printf("数字 %d 不在数组中\n", query);
    }
    
    return 0;
}
```

## 内存
我们知道计算机上CPU（中央处理器）在处理数据的时候，需要的数据是在内存中读取的，处理后的数据也会放回内存中，那我们买电脑的时候，电脑上是8GB/16GB/32GB等，是把内存划分为⼀个个的内存单元，每个内存单元的大小取1个字节。
1Byte（字节） = 8bit
1KB = 1024Byte
1MB = 1024KB
1GB = 1024MB
1TB = 1024GB
1PB = 1024TB
>“电脑 16GB 内存”“MCU 32KB 内存” → 都是 RAM。
RAM（内存） = 程序运行时使用，用来存放变量、堆栈、缓存等
掉电后数据会消失

在计算机中我们把内存单元（字节）的编号也称为地址。C语言中这个内存单元的编号，这个地址他就叫**指针**。
所以指针=地址，定义一个指针变量就是为了存储地址
每个地址可以存放一个字节的数据，
计算机内是有很多的硬件单元，⽽硬件单元是要互相协同⼯作的。所谓的协同，⾄少相互之间要能够进⾏数据传递。为了进行数据传递，引出 **“线”** 的概念。
![alt text](image.png)
| 特性 | 数据总线 (Data Bus) | 地址总线 (Address Bus) | 控制总线 (Control Bus) |
|------|---------------------|------------------------|------------------------|
| 功能 | 传输实际数据 | 传输内存/设备的地址信息 | 传输控制信号和时序信号 |
| 传输方向 | 双向（可读可写） | 单向（CPU→内存/设备） | 多为单向（部分双向） |
| 位数影响 | 决定每次传输的数据量 | 决定最大寻址空间 | 决定控制信号的丰富程度 |
| 典型宽度 | 8/16/32/64位 | 16/20/32/36位等 | 多条独立信号线 |
| 举例信号 | D0-D63 | A0-A35 | READ、WRITE、IRQ、RESET等 |

我们可以简单理解，32位机器有32根地址总线，每根线只有两态，表⽰0,1【电脉冲有⽆】，那么⼀根线，就能表⽰2种含义，2根线就能表⽰4种含义，依次类推。32根地址线，就能表⽰2^32种含义，每⼀种含义都代表⼀个地址。
**地址信息被下达给内存，在内存上，就可以找到该地址对应的数据，将数据在通过数据总线传⼊CPU内寄存器。**

# 联合和枚举
## 联合体
联合体（union）是 多个成员共享同一段内存的一种数据结构。
联合提的特点：所有成员共用同一快内存空间。
**联合体的大小是最大的成员的大小**
```C
#include <stdio.h>

//联合体类型的声明
union Un
{
    char c;
    int i;
};

int main()
{
    // 联合变量的定义
    union Un un = {0};
    printf("%d\n", sizeof(un));
    return 0;
}
```
输出结果是4，以最大成员int占的字节来看。

### 联合体变量的定义
```c
union Un un = {0};
```
初始化规则：
{0} 会初始化第一个成员（即 c = 0）
由于是初始化器，整个联合体会被清零
→ c = 0
→ i = 0
这是嵌入式中最推荐方式，因为保证所有字节为 0。
**但是这种默认是初始化第一个元素**，如果想初始化非默认元素，操作如下：
```c
union Un un = { .i = 100 };
```
表示：
- 按成员名 .i 初始化
- 其他字节自动清零
**注意：联合体初始化只能初始化一个成员！！！，因为他们是共享内存**
### 联合体变量的特点
```c
#include <stdio.h>

//联合体类型的声明
union Un
{
    char c;
    int i;
};

int main()
{
    // 联合变量的定义
    union Un un = {0};
    printf("%p\n", &(un.i));
    printf("%p\n", &(un.c));
    printf("%p\n", &un);
    return 0;
}
```
这个三个printf的输出会一样，说明不管你是取un内部的哪个元素，还是说取整个un都是一样的地址——**符合共用一块内存的特点**
**联合体内部所有成员共享同一段内存，因此它们的地址完全一致。**
内存布局如下：
```
地址 →
+----+----+----+----+
| 00 | 00 | 00 | 00 |
+----+----+----+----+
 ^       i 占用全部 4 字节
 |
 c 占用这里的 1 字节
```

**写入与覆盖现象**
```c
#include <stdio.h>

//联合体类型的声明
union Un
{
    char c;
    int i;
};

int main()
{
    // 联合变量的定义
    union Un un = {0};
    un.i=0x11223344;
    un.c=0x55;
    printf("%x\n", un.i);
    return 0;
}
```
你赋值顺序：
   -  un.i = 0x11223344;
 - un.c = 0x55; —— 写入 低字节

联合体共享同一内存，因此：
✅ 写 c 会覆盖 i 的最低字节
所以：
0x11223344 的最低字节 = 0x44
被改成了 0x55
==**最终输出**==
如果是 **小端序（Intel/ARM 默认）：**
>内存低地址存低位：
44 33 22 11
写 un.c = 0x55 → 覆盖第 0 字节：
55 33 22 11
所以 un.i = 0x11223355
✅ 大部分电脑上运行就是：
11223355


**如果是 大端序（高地址存低位）：**
>内存：
11 22 33 44
重写第 0 字节为 0x55：
55 22 33 44
所以 un.i = 0x55223344

>**这边解释一下为什么11为一组，22为一组，33为一组，44为一组这样**
**int是四个字节，一个字节存8个bit，0x代表16进制，16就是0到F一共16个数字，八个bit，每4个bit能存2^4个数也就是16个数，那么正好一位16进制可以用4个bit表示，所以1代表4个bit，11代表八个bit，所以2个16进制数等于8个bit，也就是1个字节**

>**这边解释一下小端和大端：**
**直接理解为假设数字是12345：
小端存就是倒着：54321
大端存就是正着：12345**
其实你这么想，十进制里面234，2是百位，3是十位，4是个位，显示大端就是代表大数字，那么代表百位的2是大端位置吧，那么大端存是不是高地址在前就是2在前，小端存就是低地址在前就是4在前

### 联合体大小的计算
- 联合体的大小至少是最大成员的大小。
- 当最大成员大小不是最大对齐数的整数倍的时候，就要对齐到最大对齐数的整数倍。
```c
union U {
    char  c;        // 大小 1，对齐 1
    short s;        // 大小 2，对齐 2
    char  arr[3];   // 大小 3，对齐 1
};
```
此时最大成员大小是3字节，但是他的数据类型是char——1字节，利用数组的特性（存第一个元素的字节）
那么此时要对齐到最大对齐数的整数倍，也就是short的整数倍就是3对齐到2的整数倍就是4。

# 预处理
## 预定义符号
 预定义符号是 C 语言在预处理阶段由编译器自动定义的宏
- 无需 #define
- 在预处理阶段展开
- 用来获取 编译时信息
  
| 预定义符号 | 含义 |
|-----------|------|
| `__FILE__` | 当前源文件名——例如:main.c |
| `__LINE__` | 当前行号——例如：307 |
| `__DATE__` | 编译日期 |
| `__TIME__` | 编译时间 |
| `__STDC__` | 是否遵循 ANSI C 标准 |

```C
printf("file:%s line:%d\n", __FILE__, __LINE__);
```
在程序运行时打印出当前源文件名和行号，常用于调试和日志定位。
✅ 错误处理 / 断言
```c
if (ptr == NULL) {
    printf("Error at %s:%d\n", __FILE__, __LINE__);
}
```
✅ 日志系统
```c
#define LOG(msg) \
printf("[LOG] %s:%d %s\n", __FILE__, __LINE__, msg)
```

## #include
`#include<> `——系统指定的目录——编译器内置的头文件
`#include""`——先在当前目录找，找不到去系统指定目录——自定义的头文件

## #define定义常量
基本语法：
```c
#define name  stuff
```
👉 **name——宏名**（被替换的名字）
本质：一个标识符
作用：在代码中出现 name，就会被替换成 stuff
命名规则：
符合 C 标识符规则
通常用全大写（约定）

👉 **stuff——替换内容**（宏体）
可以是：
- 数字
- 表达式
- 代码片段
- 关键字
**不做语法检查
不做类型检查
只是原样文本替换**
举例子：
```c
#define MAX 1000

#define reg register          // 为 register 这个关键字，创建一个简短的名字
#define do_forever for(;;)    // 用更形象的符号来替换一种实现
#define CASE break;case       // 在写 case 语句的时候自动把 break 写上,定义“break；case”为CASE

// 如果定义的 stuff 过长，可以分成几行，
// 除了最后一行外，每行的后面都加一个反斜杠（续行符）

#define DEBUG_PRINT printf("file:%s\tline:%d\t" \
                           "date:%s\ttime:%s\n", \
                           __FILE__, __LINE__,     \
                           __DATE__, __TIME__)
```
**// 如果定义的 stuff 过长，可以分成几行，
// 除了最后一行外，每行的后面都加一个反斜杠（续行符）**

这边解释一下`#define CASE break;case`=对于这个的理解其实就是你看到CASE把他直接替换成break；case就行——注意：C语言中；后面可以看成换行：
```c
switch (x)
{
case 1:
    foo();
CASE 2:
    bar();
}
```
等于：
```c
switch (x)
{
case 1:
    foo();
    break;
case 2:
    bar();
}
```
**#define**定义完不要加；（分号）不然会把；也作为替换的一部分

## #define定义宏
```c
#define name( parament-list ) stuff
```
函数式宏（带参数的宏） 的标准定义形式。
表示：定义一个带参数的宏
name：宏名
parameter-list：参数列表（逗号分隔）
stuff：宏展开后的文本（替换内容）
⚠️ 注意：
name 和 ( 之间**不能有空格**，否则就不是函数宏了。
举例：
```c
#define ADD(a,b) ((a)+(b))
```
```c
#define SQUARE( x )  x * x
```
表示你把SQUARE（5）置于程序中，程序会自动展开成5*5去理解，当然上面这个代码其实存在问题：
```c
int a = 5;
printf("%d\n" ,SQUARE( a + 1) );
```
乍⼀看，你可能觉得这段代码将打印36，事实上它将打印11，为什么呢？
替换⽂本时，参数x被替换成a+1,所以这条语句实际上变成了：
```c
printf ("%d\n",a + 1 * a + 1 );
```
在宏定义上加上两个括号，这个问题便轻松的解决了：
```c
#define SQUARE(x) (x)*(x);
```
>所以⽤于对数值表达式进⾏求值的宏定义都应该⽤这种⽅式加上括号，避免在使⽤宏时由于参数中的
操作符或邻近操作符之间不可预料的相互作⽤。
在式子外面加大括号：
```c
#define SQUARE(x) ((x)*(x));
```
## 宏替换的规则
在程序中扩展#define定义符号和宏时，需要涉及⼏个步骤。
1. 在调⽤宏时，⾸先对参数进⾏检查，看看是否包含任何由#define定义的符号。如果是，它们⾸先
被替换。
2. 替换⽂本随后被插⼊到程序中原来⽂本的位置。对于宏，参数名被他们的值所替换。
3. 最后，再次对结果⽂件进⾏扫描，看看它是否包含任何由#define定义的符号。如果是，就重复上
述处理过程。

**注意：**
1. 宏参数和#define定义中可以出现其他#define定义的符号。但是对于宏，**不能出现递归。**
   不允许如下：
   ```c
    #define A B
    #define B A
    ```
2. 当预处理器搜索#define定义的符号的时候，字符串常量的内容并不被搜索。
   ```c
    #define HELLO world

    printf("HELLO\n");
   ```
   这里依然是打印出来HELLO不打印world

## 宏和函数的对比
宏有时候可以做函数做不到的事情。⽐如：宏的参数可以出现**类型**（int、char这些），但是函数做不到。
```c
#define MALLOC(num, type)\
(type*)malloc(num  sizeof(type))
...
//使⽤
MALLOC(10, int);//类型作为参数

//预处理器替换之后：
(int *)malloc(10  sizeof(int));
```
